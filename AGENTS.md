# Agent Guidelines for skill-gen

This repository defines the architecture for generating autonomous, high-impact AI agent skills. Every modification must adhere to these principles.

## Core Architecture: "God Tier" Skills

Skills generated by `skill-gen` must not be mere task-runners. They must be **architectural blueprints** for agents that can operate at scale (city-wide impact).

1.  **Fault Tolerance:** Every step must include a "Failure Mode" and a "Self-Healing" path.
2.  **Parallelism:** Use delegation (`Agent` or `worker`) for independent sub-tasks.
3.  **Validation-First:** Verify system state *before* and *after* every destructive action.
4.  **Zero Hallucination:** Instructions must mandate reading from disk or shell before making assumptions.
5.  **Terse Communication:** Generated skills must output only progress and final outcomes.

## File Organization

- `SKILL.md`: Entry point and orchestration.
- `interviewer.md`: Intelligence gathering and user confirmation logic.
- `builder.md`: The scaffold engine and file generation rules.
- `templates/`: Platform-specific markdown skeletons.
  - `claude-code.md`: For Claude Code (uses `Agent`, `Bash`, `Read`, `Write`).
  - `codex.md`: For Codex CLI (uses `update_plan`, `apply_patch`, `shell`).
  - `universal.md`: Agnostic, tool-neutral fallback.

## Coding Standards for Markdown

- **No Placeholders:** Final generated skills must have zero `{{placeholders}}`.
- **Absolute Paths:** Generated shell commands should prefer absolute or workspace-relative paths.
- **Regex Safety:** All user-provided names must be sanitized via `^[a-z0-9-]+$`.
- **No Dependencies:** Keep `skill-gen` zero-dependency. Do not add Python or Node helper scripts.

## Quality Gate Checklist

When modifying templates or the builder:
- [ ] Does this allow for city-scale automation?
- [ ] Is the logic resilient to partial failures?
- [ ] Is the output concise and actionable?
- [ ] Is there a clear escape hatch for the user?
